Laravel Upgrade Assistant

Scan your codebase for compatibility issues before you upgrade. Detects only the breaking changes that actually exist in your project — not a generic checklist.

PHP ^7.4 | ^8.0 Laravel 8 – 13 MIT License Auto-Discovery
$ composer require jaydeep/laravel-preflight

⚡ Installation

Install the package via Composer. No manual service provider registration is needed — Laravel's package auto-discovery handles it.

composer require jaydeep/laravel-preflight

Once installed, the upgrade:check Artisan command is immediately available.

🚀 Usage

Run the command with an optional target version. If no version is specified, the latest supported version (Laravel 13) is used automatically.

bash# Check against the latest version (Laravel 13)
php artisan upgrade:check

# Check against a specific target version
php artisan upgrade:check 9
php artisan upgrade:check 10
php artisan upgrade:check 11
php artisan upgrade:check 12
php artisan upgrade:check 13

# Save a full Markdown report to storage/upgrade-report.md
php artisan upgrade:check 11 --report

The command reads your composer.json to detect the current Laravel version automatically. Multi-version upgrades are fully supported — checking from Laravel 8 to 13 will scan all intermediate breaking changes (9, 10, 11, 12, and 13).

🔍 Analyzers

Four independent analyzers run in sequence, each responsible for a different area of your project.

📦
ComposerAnalyzer
Scans composer.json require and require-dev for deprecated or removed packages.
🔎
CodeAnalyzer
Regex-scans all .php files in app/, routes/, config/, database/, resources/.
⚙️
ConfigAnalyzer
Checks config/*.php files for missing or renamed configuration keys.
🌍
EnvAnalyzer
Checks .env.example (or .env) for new required environment variables.

📅 Supported Versions

The following upgrade paths are tracked. Checking from a lower version to a higher one automatically includes all intermediate steps.

LevelMeaningAction Required

📄 Report File

Use the --report flag to generate a Markdown report saved to storage/upgrade-report.md.

php artisan upgrade:check 13 --report

The report contains:

  • Summary table — critical / warning / info counts and date
  • Every detected issue with affected file paths and fix instructions
  • Full table of all checks that passed

🔧 Adding a New Version

Three steps to add support for a new Laravel version:

1. Create the upgrade class

PHPclass Laravel14Upgrade extends BaseUpgrade
{
    public function getBreakingChanges(): array
    {
        return [
            $this->change(
                title:         'SomeClass removed',
                description:   'SomeClass was removed in Laravel 14.',
                severity:      'critical',
                category:      'code',
                fix:           'Replace SomeClass with NewClass.',
                searchPattern: '/SomeClass/',
            ),
        ];
    }
}

2. Register in VersionRegistry

PHPprivate $upgrades = [
    '9'  => Laravel9Upgrade::class,
    '10' => Laravel10Upgrade::class,
    '11' => Laravel11Upgrade::class,
    '12' => Laravel12Upgrade::class,
    '13' => Laravel13Upgrade::class,
    '14' => Laravel14Upgrade::class, // ← add here
];

private $latestVersion = '14'; // ← bump this
